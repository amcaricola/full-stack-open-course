sequenceDiagram
participant user
participant browser
participant server

    user->>browser: writes a note in the field and press the "Save" button

    browser->>server: does a POST request to https://studies.cs.helsinki.fi/exampleapp/new_note_spa

    activate server
    Note right of browser: The browser sends a JSON of the new content "[{ "content": "my new note", "date": "2026-02-17" }]"

    server-->>browser: Process the data, and sends a reply
    deactivate server

    Note left of server: The server saves the new note internally and reply the petition with the "same data"

    Note right of browser: the spa.js handles the event, it creates internally the HTML and then tells the browser to render it

    browser-->>user: renders the new note without refreshing the page

    Note right of user: Now user can see the new note
